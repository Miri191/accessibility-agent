# בדיקות נגישות - מדריך מקיף

## סוכן בדיקת נגישות - פרמטרים שנבדקים

### ✅ בדיקות בסיסיות (Level A)

1. **שפת הדף (lang attribute)**
   - בודק שתג HTML מכיל תכונת lang
   - דרישה קריטית לקוראי מסך

2. **כותרות (Headings Hierarchy)**
   - בודק קיום h1 בדף
   - בודק שאין דילוג ברמות כותרות (למשל קפיצה מ-h2 ל-h4)
   - בודק שיש רק h1 אחד בדף

3. **טקסט חלופי לתמונות (Alt Text)**
   - בודק שכל התמונות בדף מכילות טקסט alt
   - מזהה תמונות ללא טקסט חלופי

4. **קישורים (Link Text)**
   - בודק שלקישורים יש טקסט תיאורי
   - מזהה טקסטים גנריים כמו "לחץ כאן", "קרא עוד"
   - בודק aria-label כאלטרנטיבה

### 🔵 בדיקות מתקדמות (Level AA)

5. **תוויות לשדות טופס (Form Labels)**
   - בודק שכל שדה input/textarea/select מקושר ל-label
   - תומך ב-aria-label ו-aria-labelledby
   - בודק גם labels שעוטפים את השדה

6. **כפתורים (Buttons)**
   - בודק שלכפתורים יש טקסט נגיש
   - בודק value, text content, ו-aria-label

7. **טבלאות (Tables)**
   - בודק שטבלאות מכילות כותרות (<th>)
   - בודק תכונות scope ו-headers

### 🎯 בדיקות מבניות

8. **ARIA Landmarks**
   - בודק קיום אזורי ניווט עיקריים:
     - main (תוכן עיקרי)
     - nav (ניווט)
     - footer (כותרת תחתונה)
   - עוזר למשתמשי קוראי מסך לדלג למקומות חשובים בדף

9. **קישורי דילוג (Skip Links)**
   - בודק קיום קישור "דלג לתוכן הראשי"
   - חשוב למשתמשי מקלדת

### 🎨 בדיקות ויזואליות (בסיסי)

10. **ניגודיות צבעים (Color Contrast)**
    - בדיקה בסיסית של סגנונות inline
    - **הערה**: בדיקה מלאה דורשת ניתוח CSS ורינדור
    - מומלץ להשתמש בכלים מבוססי דפדפן לבדיקה מקיפה

### 📊 סטטיסטיקות שנאספות

הסוכן אוסף סטטיסטיקות מפורטות:
- מספר תמונות כולל / תמונות ללא alt
- מספר קישורים / קישורים לא ברורים
- מספר שדות טופס / שדות ללא labels
- מספר כפתורים / כפתורים ללא טקסט
- מספר טבלאות / טבלאות ללא כותרות

### 🎖️ רמות WCAG

הסוכן מחשב אוטומטית:
1. **רמה שהושגה** - A, AA, AAA, או לא עומד בתקן
2. **רמה נדרשת** - על פי סוג האתר:
   - ממשלתי: AAA
   - בנקאות/פיננסים: AA
   - שירותים ציבוריים/בריאות/חינוך: AA
   - מסחר אלקטרוני: A
   - כללי: A (מומלץ)

3. **סטטוס עמידה** - מציג בצורה ויזואלית האם האתר עומד בכל רמה

## מגבלות ובדיקות עתידיות

### 🔴 בדיקות שדורשות דפדפן

הבדיקות הבאות דורשות כלים מבוססי דפדפן:
- **ניגודיות צבעים מלאה** - צריך לנתח CSS ולחשב צבעים מחושבים
- **גודל טקסט** - דורש רינדור
- **Keyboard navigation** - צריך סימולציה של מקלדת
- **Focus indicators** - דורש רינדור ואינטראקציה
- **Video/Audio** - בדיקת כתוביות וטרנסקריפטים

### ✨ בדיקות נוספות שכדאי להוסיף

- **עיצוב רספונסיבי** - בדיקת viewport meta tag
- **פונטים ניגשים** - גודל מינימלי, font-family
- **אנימציות** - prefers-reduced-motion
- **טפסים** - autocomplete, error messages
- **ARIA roles** - בדיקה מקיפה של ARIA

## המלצות לשימוש

1. **הרץ את הבדיקה** על כל דף חדש לפני פרסום
2. **עקוב אחר הבעיות הקריטיות** תחילה
3. **השתמש בכלים משלימים** לבדיקת ניגודיות מלאה
4. **בדוק עם קוראי מסך** אמיתיים (NVDA, JAWS, VoiceOver)
5. **בדוק ניווט במקלדת** ידנית

## כלים משלימים מומלצים

- **axe DevTools** - בדיקה מקיפה בדפדפן
- **WAVE** - הדגמה ויזואלית של בעיות
- **Lighthouse** - בדיקת ביצועים ונגישות
- **Color Contrast Analyzer** - בדיקת ניגודיות מדויקת
- **NVDA / JAWS** - טסט עם קוראי מסך אמיתיים
